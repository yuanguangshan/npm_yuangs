# 微信读书主题研究

## 概述

给定一个主题关键词，自动完成"搜索 → 筛选高分书 → 获取详情 → 抓取优质评论 → 热门划线 → 整合成深度文章"的完整流程。默认选 5 本，用户可指定数量。

## 你的工作方式

这是一个**流程型**技能，串行执行 5 个阶段。核心 API 调用都封装在 `scripts/research.py`，你只需正确调用脚本并整合输出，不要在对话中手撕 curl。

**关键技术约定（已踩过坑，必须遵守）**：

1. **WEREAD_API_KEY 必须 export**：环境变量不会自动注入，从 memory 取值或让用户提供，每次新 shell 都要重新 export。
2. **API 请求体 params 平铺**：`/book/info`、`/review/list` 等接口的参数（如 `bookId`）必须直接放在 JSON 根层级，**不要嵌套在 `params` 对象内**，否则报 `errcode:-2003 缺少必填参数`。
3. **必须带 `skill_version: "1.0.3"`**：缺这个字段会鉴权失败。
4. **用 curl 不用 python urllib**：urllib 调用此 API 偶发 SSL 错误，curl 稳定。
5. **research.py 已封装以上所有坑**，正常情况直接跑脚本即可。

## Phase 1：解析用户输入

从用户消息中提取：
- **关键词**（必填）：主题词，例如"大模型""量化交易""认知科学"
- **数量**（可选，默认 5）：用户若说"选 3 本""挑 10 本"则取该数字，否则用 5
- **每本评论数**（可选，默认 3）：用户若特别强调"多放点评论"可调到 5

不要追问用户。能从对话直接提取就直接跑。

## Phase 2：执行研究脚本

```bash
export WEREAD_API_KEY=<从 memory 或环境获取>
python3 /sandbox/workspace/skills/weread-topic-research/scripts/research.py \
  --keyword "<关键词>" \
  --top-n 5 \
  --reviews-per-book 3 \
  --output /sandbox/workspace/outputs/weread_research_<关键词>_<YYYYMMDD>.md
```

脚本输出：
- 完整的 Markdown 文章到指定路径
- stdout 打印进度（搜索到 N 本 → 筛选 top 5 → 抓取详情/评论 → 写入文件）
- 返回文章字符数和文件路径

如果脚本报错，先看错误类型再决定：
- `WEREAD_API_KEY 为空` → 让用户提供或从 memory 取
- `搜索结果为空` → 关键词太冷门，建议用户换词或拓宽
- `部分书无评论` → 脚本会自动跳过，输出不受影响
- 其他错误 → 读 stderr，按真实原因修，不要重试相同参数

## Phase 3：质量检查与微调

读取生成的 Markdown，自查：

- [ ] 5 本书是否都包含：书名/作者/评分/简介/精选评论
- [ ] 评论是否有信息密度（避免"好书推荐"这种水评）
- [ ] 开篇是否有主题导读（说明为什么这 5 本能代表此主题）
- [ ] 结尾是否有延伸视角（推荐阅读顺序 / 适合人群 / 后续主题）

如果脚本生成的内容偏机械（仅简单拼接），用 `file_edit` 在开篇和结尾补充人类视角的导读与收束。**不要重写已有的书籍介绍和评论原文**——评论是用户的原创表达，保真比润色更重要。

## Phase 4：可选 · 分发

**默认不自动分发**——文章生成到 workspace 后即结束流程。

仅当用户在请求中**明确**说了"发到微信""推到博客""上传 knowly"等分发动作时，才执行对应分发：

- 发微信 → 触发 `wechat-send` skill
- 发博客 → 触发 `blog_publisher` skill
- 上传 knowly → 触发 `上传到knowly` skill

不要把分发当成主流程的一部分主动做。

## Phase 5：交付报告

回复用户时给出：
- 文章保存路径（workspace 路径）
- 字符数 / 选了哪几本（带评分）
- 一句话主题点评（你对这个主题书单的整体判断）
- 提示用户："如需发到微信/博客/knowly，告诉我即可"

**不要在回复中粘贴整篇文章**——文章在 workspace 可读，回复保持简洁。

## 使用示例

**示例 1：标准请求**
> 用户："微信读书研究一下'宏观经济'这个主题"
>
> 执行：脚本默认 5 本 3 评 → 生成文章 → 发微信 → 回复摘要

**示例 2：自定义数量**
> 用户："帮我搜量化交易相关的书，选 8 本，每本多放几条评论"
>
> 执行：`--top-n 8 --reviews-per-book 5` → 后续同上

**示例 3：连续主题**
> 用户："再来一个'认知科学'的"
>
> 执行：直接复用流程，关键词换成"认知科学"
